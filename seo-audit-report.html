1.3 高优先级 - 非 JSON-LD 的 url 字段在非英语页面指向英文根

Organization 与 WebSite schema 的 url 在 /ja 页面仍指向英文根 URL

高 High 影响：中 结构化数据
问题
src/app/[locale]/layout.tsx 第 53 行，Organization schema 的 url 与 logo 都使用 siteUrl（英文根）：
const organization = { "@type": "Organization", name: messages.site.name, url: siteUrl, // ❌ 永远指向英文根 ... };
src/app/[locale]/page.tsx 第 32 行的 WebSite schema 同样问题：url: siteUrl。
抓取 /ja 主页验证：Organization.url = "https://dead-or-alive-6-last-round.wiki"（非 /ja）。
影响与修复
影响：所有 locale 的 schema 都把英文 URL 当作主体 URL，混淆了 Google 对实体与页面关系的理解。在多语言站点中，schema 的 url 应指向当前页面所属语言的规范 URL。

修复：在 Organization 与 WebSite schema 中使用本地化 URL：
const localizedUrl = locale === "en" ? siteUrl : `${siteUrl}/${locale}`; const organization = { "@type": "Organization", name: messages.site.name, url: localizedUrl, // ✅ 本地化 logo: `${siteUrl}/android-chrome-512x512.png`, ... };
注意：logo 与 image 这类资源 URL 可以保持全局一致，只需 url 本地化。

1.4 高优先级 - localeDetection 自动重定向可能干扰抓取

localeDetection: true 基于 Accept-Language 自动重定向

高 High 影响：中 国际 SEO
问题
src/i18n/routing.ts 第 16 行：localeDetection: true。
验证：访问 http://localhost:3001/ 时，由于浏览器 Accept-Language 头被识别为日语环境，被重定向到 /ja。
// 浏览器导航日志： Page URL: http://localhost:3001/ja // 实际请求的是 /
虽然 Googlebot 通常以 US IP + 无 Accept-Language 抓取（会得到 en），但其他爬虫/工具可能触发重定向，且本地用户首次访问时被强制重定向可能影响 backlink 与分享链接的目标一致性。
影响与修复
影响：Google 明确不建议基于 IP/Accept-Language 的内容协商，因为隐藏了内容且造成 URL 不稳定。

修复：将 localeDetection 设为 false，改用明显的语言切换器（站点已有 language-switcher.tsx）+ 清晰的 hreflang 让 Google 自行匹配：
export const routing = defineRouting({ locales: ["en", "ja", "ko", "es"], defaultLocale: "en", localePrefix: "as-needed", localeDetection: false, // ✅ 关闭自动检测 });

1.5 中优先级 - 图片无 loading="lazy"

所有页面图片均未启用懒加载

中 Medium 影响：低-中 性能
问题
抓取结果：主页 2 张图片，详情页 1 张图片，0 张带 loading="lazy"。
